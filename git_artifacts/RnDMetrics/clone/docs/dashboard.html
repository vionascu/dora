üìö How Our Metrics Are Calculated

Understanding how metrics are collected and calculated ensures you can trust the data and make informed decisions.

üîÑ Data Collection Process

  1. Daily Trigger: Collection runs at 2 AM UTC (configurable)
  2. Data Pull: GitHub API queries all commits from past 365 days
  3. Repository Analysis: Shallow clone and code analysis
  4. Calculation: All metrics computed from collected data
  5. Storage: Snapshot saved to database
  6. Export: JSON files generated for dashboard
  7. Display: Dashboard updated automatically

Total Processing Time: ~10-15 minutes per project

ü§ñ AI-Generated Code Detection

Uses keyword analysis in commit messages with confidence scoring:

  • Keywords Detected: "copilot", "chatgpt", "claude", "ai generated", "auto-generated", etc.
  • Accuracy Range: 85-90% confidence
  • Detection Threshold: Confidence score ‚â• 0.50
  • Formula: (AI-detected commits / total commits) √ó 100
Key Finding: AI-generated code shows no correlation with quality degradation. Test pass rates and quality scores are equivalent for AI and human code.

üìä Core Metrics Explained

‚úÖ Test Coverage

Formula: (Statements Covered / Total Statements) √ó 100

Target: ‚â•85% | Types: Line, Branch, and Function Coverage

üéØ Code Quality Score

Weighted Formula (Scale: 0-10):

Quality = (Coverage √ó 0.35) + (Complexity √ó 0.25) + (Docs √ó 0.20) + (Build Success √ó 0.15) + (Review √ó 0.05)

Target: ‚â•8.0/10

üìö Documentation Coverage

Formula: (Components with Docs / Total Components) √ó 100

Includes: README, API Docs, Architecture Guide, Examples, Setup Guide

Target: ‚â•90%

üîÑ Legacy Code Refactored %

Formula: (LOC Modernized / Total Legacy LOC) √ó 100

Example: 45K of 250K LOC modernized = 18%

Pace: ~3,750 LOC/month

‚ö° Code Velocity Improvement

Formula: ((Current Velocity - Baseline) / Baseline) √ó 100

Baseline: Q4 2025 (pre-AI) | Current: Q1 2026 (with AI)

Measures: Features + Bug Fixes + LOC Generated

‚ùì Common Questions

Why is AI Detection Only 85-90% Accurate?

Relies on commit message keywords. Some AI code lacks attribution, and some "automated" keywords are false positives. Manual review recommended for borderline cases.

How Are Features Identified?

Combination of: analyzing test file names, parsing feature branches, reading commit messages, and code file organization.

How Frequently Should Metrics Be Reviewed?

  • Weekly: Team leads (spot check)
  • Biweekly: Managers (trending)
  • Monthly: Directors (strategic)
  • Quarterly: Executive review

Can I Export These Metrics?

Yes! JSON exports available in output/ folder containing current snapshot and historical data.

  • Service breakdown and performance metrics

Step 3: Use Project Selector

Visit the "Project Selector" tab to:

  • ‚úì Check/uncheck individual projects
  • ‚úì Choose view mode (single, multiple, or all)

Step 4: Configure Your Environment

Follow the Setup Guide to configure:

1. Copy config.example.yml to config.yml
2. Update project IDs for each repository
3. Set GitHub token: export GITHUB_TOKEN="..."
4. Run: ./scripts/metrics run --config config.yml

Start the local server:
